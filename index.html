1. 기본 사용 흐름

① 리뷰리포트(가리뷰 참여자 명단)와 주문 파일(확장주문검색·스마트스토어 발주발송관리 등 자동 인식)을 업로드 → ② [누적 분석] 클릭 → ③ 결과가 실주문 / 가리뷰 / 리뷰참여자 탭으로 분리 → ④ 작업 완료한 건은 완료 체크박스로 표시.

지원 파일: 헤더에 수취인 또는 수령자가 들어간 어떤 형식이든 자동 인식. 네이버 스마트스토어처럼 비밀번호 0000이 걸려 있어도 자동 해제됩니다.

2. 매일 신규건 처리

매일 새 주문 파일이 들어오면 그 파일만 다시 업로드 → [누적 분석]. 기존 누적은 그대로 유지되고 NEW 배지가 붙은 행만 새로 추가됩니다. 리뷰리포트가 업데이트되면 함께 업로드 — 누적된 모든 주문에 대해 가리뷰 여부가 자동 재판정됩니다.

3. 오늘/어제 구분

각 주문에는 처음 업로드된 날짜(_수집일자)가 기록됩니다. 상단 날짜 필터로 오늘/어제/최근 7일/전체를 골라 볼 수 있습니다.

4. 완료 처리 (3명 실시간 공유)

행 첫 컬럼의 체크박스를 클릭하면 즉시 서버에 저장되고, 다른 PC에서 5초 안에 자동 반영됩니다. 메모 칸에 송장번호·작업메모를 적어도 동일하게 공유됩니다.

5. 백업 / 복원

[백업 다운로드]로 누적 데이터를 JSON 파일로 저장하세요. [백업 복원]은 복원 파일과 같은 주문번호가 있으면 덮어쓰지 않고 신규만 병합합니다.

6. 매칭 키 변경

기본은 전화 끝 4자리. 끝자리 우연 충돌이 우려되면 전화 끝4 + 이름2자로 바꾸고 [누적 분석]을 다시 누르면 모든 행이 재평가됩니다.
⚠ 운영 유의사항
· 데이터는 3명이 공유합니다. [전체 초기화]는 모두에게 영향. 신중히.
· 같은 주문번호의 신규 업로드는 덮어쓰지 않고 무시됩니다(완료/메모 보존). 강제 재처리는 [전체 초기화] 후 다시 업로드.
· 5초마다 자동 동기화됩니다. 다른 사람이 동시에 같은 행을 수정하면 마지막 변경이 이깁니다.
⚠ 보안 유의사항
· 이 URL을 아는 누구나 데이터(수령자 주소·전화 끝자리 등 개인정보 포함)에 접근할 수 있습니다. 3명 외부로 절대 공유 금지.
· 메일/카톡/문서에 URL 평문 기재 시 외부 노출 위험. 비밀 메모/패스워드 매니저 사용 권장.
· 보안 강화가 필요하면 Supabase Auth 도입을 요청하세요.
